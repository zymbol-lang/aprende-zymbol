# Aprende Zymbol — Nivel Básico

Aprender a programar no es aprender sintaxis. Es aprender a pensar de una manera que antes no usabas.

Al principio el problema no es el lenguaje — es que la computadora hace exactamente lo que le dices, sin interpretar, sin suponer, sin adivinar. Eso es nuevo. Estás acostumbrado a comunicarte con personas que llenan los huecos, que entienden el contexto, que corrigen solos cuando algo no tiene sentido. La computadora no hace nada de eso. Y adaptarse a eso toma tiempo.

Estas ocho lecciones están ordenadas para que ese ajuste sea gradual.

---

**Instalación** es la primera lección porque antes de escribir una sola línea necesitas confirmar que el entorno funciona. Parece trivial. No lo es. Muchos aprendices se atascan aquí más tiempo del que esperaban, y saber que eso es normal — que instalar herramientas es una habilidad en sí misma — ahorra frustración.

**Tu primer programa** te pone frente al flujo más básico de cualquier programa: recibir datos, procesarlos, mostrar resultados. Variables, tipos, entrada y salida. Nada espectacular todavía, pero es la base sobre la que se construye todo lo demás. Si esto queda claro, el resto avanza solo.

**Decisiones** introduce el momento en que el programa deja de ser una secuencia fija y empieza a reaccionar. La condición es la unidad mínima del razonamiento computacional: si esto es verdad, haz aquello; de lo contrario, haz esto otro. Aquí el código empieza a parecerse a lógica real.

**Selector de casos** aparece cuando las decisiones se multiplican. Dos caminos se manejan bien con una condición. Siete caminos, doce, veinte — ahí se necesita una estructura diferente. El selector de casos no es una variante de las decisiones: es una herramienta distinta para un problema distinto, y reconocer cuándo usar cada una es parte del oficio.

**Repetición** es donde el programa deja de ser manual. Hasta este punto, cada instrucción tiene que estar escrita explícitamente. Los bucles cambian eso: defines una acción una sola vez y el programa la repite cuantas veces haga falta. Es el primer momento en que el código hace más trabajo del que tú escribiste.

**Listas** resuelven el problema de los datos múltiples. Hasta ahora cada variable guarda un valor. Las listas guardan colecciones — y junto con la repetición, forman el par más usado en cualquier programa real. Recorrer una lista con un bucle es el patrón que aparece en casi todo lo que escribirás de aquí en adelante.

**Texto** amplía lo que puedes hacer con cadenas de caracteres. No es una lección de detalles menores — el texto es el tipo de dato con el que más interactúan los usuarios, y Zymbol ofrece herramientas específicas para trabajarlo con comodidad: interpolación, comparación, subcadenas, caracteres.

**Funciones** es la lección que cambia cómo piensas el código, no solo cómo lo escribes. Hasta aquí has estado resolviendo problemas. Aquí empiezas a nombrar soluciones. Una función es un bloque con nombre, con entradas y salida definidas, que puedes llamar desde cualquier parte. El código que viene después de esta lección es cualitativamente diferente: más corto, más claro, más fácil de modificar.

---

Al terminar estas ocho lecciones tendrás las herramientas para escribir programas completos que hacen cosas reales. No serán simples ni triviales — serán programas con lógica, con datos, con estructura.

El [proyecto final](basico/09_proyecto.md) te propone construir uno de principio a fin. Los [ejercicios](basico/ejercicios.md) te dan práctica progresiva lección por lección.
